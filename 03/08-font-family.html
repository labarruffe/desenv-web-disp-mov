Famílias de fontes

Há diferentes famílias de fontes, que são usadas para diferentes identidades visuais. É importante, mesmo que a fonte original falhe por algum motivo (download, CORS) que a sua substituta seja uma fonte da mesma família, para tentar manter o produto que o cliente está revebendo o mais próximo possível do design original. São famílias de fontes:

/* Exemplo de como manter a fonte dentro da mesma família */ .font-times-new-roman { font-family: "Times New Roman", serif; }

serif

Fontes com serifa, aquelas "barrinhas" no final das letras. Geralmente são utilizadas apenas em títulos, mas hoje em dia não há problema em utilizá-las no corpo do texto, pois a resolução e qualidade dos displays permite a fácil leitura desses tipos.

exemplos

  • Times New Roman
  • Georgia
  • Garamond
  • MS Serif
  • Repositório: https://fonts.google.com/?category=Serif

sans-serif

São fontes que não apresentam as serifas, muito adequadas para o corpo do texto, principalmente em dispositivos mobile, pois seu acabamento simples evita que detalhes pequenos (como as serifas) pareçam apenas borrões em tamanhos pequens. Tendem a ser bem flexíveis em redimensionamento, podendo ser usadas em tamanhos pequenos (8px, por exemplo) sem muitos problemas.

exemplos

  • Arial
  • Helvetica
  • Verdana
  • Montserrat
  • Repositório: https://fonts.google.com/?category=Sans+Serif

cursive

São fontes curvadas, assemelhadas a escrita à mão. Nem todas são ligadas / emendadas. São adequadas apenas para títulos, e com ressalvas, pois boa parte delas tem uma legibilidade baixa, PRINCIPALMENTE em tamanho pequeno e / ou negritadas.

exemplos

  • Comics Sans MS
  • Repositório: https://fonts.google.com/?category=Handwriting

monospace

Fontes cujos símbolos ocupam o mesmo espaço, ou seja, todas as letras têm a mesma largura. Geralmente são utilizadas para representar código de máquina e/ou linguagens de programação, já que a maioria dos editores de código usa fontes monoespaçadas para facilitar o alinhamento de diferentes linhas de código.

Alguns navegadores utilizam fontes monospace para elementos de formulário, o que pode levá-los a destoarem completamente do resto da identidade visual do projeto, requerendo, portanto, uso d e normalização do CSS para que as fontes adequadas sejam aplicadas nos elementos de formulário.

exemplos

  • Courier New
  • Repositório: https://fonts.google.com/?category=Monospace

Fontes seguras para a web (safe fonts)

São fontes muito comumente distribuídas junto com os Sistemas Operacionais, e portanto, podendo ser facilmente utilizadas sem precisarmos criar webfonts. Uma listagem delas pode ser encontrada em: http://web.mit.edu/jmorzins/www/fonts.html 

Elas são uma ótima solução para projetos pequenos, mas infelizmente são fontes antigas e pouco refinadas, sendo menos adequadas para trabalhos de alto-nível e/ou com cunho artístico; nesse caso devemos recorrer a webfonts.
