Dicas: graceful degradation

Alguns princípios do graceful degradation ainda são extremamente úteis

Planeje para a falha das fontes, teste se o layout não vai quebrar com textos maiores, e NUNCA paute a altura dos elementos pelo tamanho do texto, use height: auto ou controle de overflow.

Planeje para a falha de imagens, e teste com elas faltando, o layout não pode quebrar e veremos se os devidos atributos alt estão setados.

Planeje para a falha de carregamentos assíncronos (Ajax) e SEMPRE ofereça feedback para o usuários

Sempre use <a> para todos os elementos de navegação, e referencie URLS no href, para o site ser navegável mesmo se o JS falhar (e pode ser que na próxima página ele não falhe).

Outro motivo para usar <a> para a navegação é que na maioria dos navegadores mobile o evento click é disparado apenas para <a> e <button>, fazendo com que VÁRIOS javascripts não funcionem corretamente no mobile.

No JavaScript tu podes suprimir o comportamento padrão do elemento, como, por exemplo, o <a> levar para uma nova página, e fazer com que alguma função escrita por ti rode no lugar, criando assim a navegação que tu sonhas.
