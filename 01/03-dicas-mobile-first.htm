Dicas: mobile-first

Faça toda a versão mobile e depois adicione media-queryes para as resoluções maiores. As vantagens são:

  • O fluxo da interpretação CSS é um aliado, não um inimigo
  • Na versão mobile não expomos (para depois ocultarmos) elementos que deveriam aparecer apenas em resoluções maiores
  • Não é preciso "ir e voltar" em configurações, pois o fluxo das Media-queryes é o mesmo do CSS
  • As media-queryes serão bem mais enxutas se escritas assim
  • A manutenção é facilitada por não precisar "catar" media-queryes

Como o tamanho da tela varia muito no mobile, por exemplo de 320px a 375px apenas entre iPhones, e isso em modo Retrato Em se tratando de altura o problema é bem maior, no exemplo anterior seria 480 x 667. Já saímos preparados do design mobile para encarar qualquer outro problema, pois comaçamos jogando no "hard".

Ao contrário dos usuários desktop, a maioria dos usuários mobile estão com seus navegadores atualizados, logo, podemos utilizar todoas as "legalzices" novas que aparecem sem muito medo. Já no desktop isso é muito temerário, cito como exemplo:

  • Localização (GPS)
  • Multi-touch
  • Bluetooth
  • Aceleração por hardware (placa de vídeo)
  • Acelerômetros
  • Input de vídeo e áudio através de microfone/câmera
