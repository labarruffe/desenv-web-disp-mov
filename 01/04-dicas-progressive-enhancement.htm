Dicas: progressive-enhancement

O conteúdo, tudo o que realmente importa, precisa estar no HTML, e estar na forma mais amigável possível para o usuário caso tudo mais falhe.

Com o conteúdo onde deve estar cuide da aparência. Comece pela tipografia, depois esquema de cores, aí parta para outros aspectos da aparência (background-images, bordas, etc...).

Com tudo no lugar podemos começar a pensar na interatividade, com hovers, transitions, animations e toda e qualquer sorte de recursos.

A versão mobile estará pronta em pouco tempo, para a felicidade do cliente. E como o conteúdo está todo definido já, e não teremos a longa fase de "cortar conteúdo" ganhamos um BOM tempo.

Todo o HTML extra que não for 100% nescessário para apresentar a página, por exemplo, 1 slider de fotos, deve ser inserido usando JS.

Respeite as camadas do desenvolvimento web, e use-as de modo correto.


HTML
Deve ser o mais limpo possível, e semântico. Assim leitores de tela e crawlers (robôs da web) poderão ler o site corretamente. Em hipótese alguma defina aparência aqui (widths, heights, cores), como era prática no HTML4.
CSS
Define a forma que o site vai assumir em user-agents visuais, como desktops, telefones, tablets. Informações de tipografia, cores, tamanhos devem estar aqui. Sempre que possível devemos fazer transições eficientes do que utilizar JS para o mesmo. e animações utilizando CSS e trocas de classe, pois são muito mais rápidas do que as feitas em jQuery (teste de velocidade).
JS
Use para incrementar a experiência, não fazê-la, pois é o ponto mais comum de falha. O caso ideal é que o site funciona mesmo se nenhum JS carregar.
