Padrões de layout responsivo

Mostly Fluid


  • O Layout tem uma largura máxima, mais do que isso há acréscimo de margens.
  • Usa grids responsivos, e depende de resize de imagens em tamanhos menores de tela
  • As colunas empilham-se verticalmente em resoluções menores
  • A estrutura do layout só é modificada na menor resolução prevista
  • Exemplo: Five Simple Steps

Column drop


  • O Layout inicia com várias colunas e termina com uma única
  • Conforme a resolução diminui as colunas vão "caindo" para baixo
  • O tamanho dos elementos (imagens) tende a ser consistente
  • Exemplo: Modernizr

Layout shifter


  • Em teoria é o mundo ideal, todos os designs deveriam ser assim, adaptando os elementos para prover a melhor experiência para cada dispositivo
  • Entretanto é o layout que mais requer trabalho para ser implementado
  • A altura dos elementos também é importante, o que aparece na tela em cada momento é importante
  • Há muita variação na implementaçõa para ser facilmente representado, ao contrário dos layouts anteriores
  • Em suma, em cada breakpoint temos um design diferente
  • Exemplo: The Boston Globe

Tiny Tweaks


  • O simples é belo.
  • Apenas possível para sites com layout de uma coluna
  • Apenas ajustes de tamanho de imagens e vídeos são nescessários, se eles forem responsivos, nenhum ajuste é nescessário.
  • Exemplo: Future Friendly

Off canvas


  • Design estilo app de mobile.
  • Elementos do layout estão fora da tela, sendo trazidos para a tela através de interações do usuário
  • Passa a idéia de que em resoluções maiores esses elementos estarão na tela
  • Exemplo: Disney
