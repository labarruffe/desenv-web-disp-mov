Design Responsivo

Um conjunto de técnicas e práticas para fazer com que a tua página seja apresentada da melhor maneira possível em qualquer resolução, utilizando apenas um conjunto de arquivos (seja HTML e/ou CSS) para isso.

Principais técnicas e práticas

Media-queryes
Como aplicar blocos de regras CSS apenas em algumas situações (resoluções, DPIs, mídias)
GRIDS
Estrutura básica do site em forma de grid (conjunto de linhas e colunas), assemelhado ao layout de páginas em papel, mas baseado em %. É genérico, e não baseado no projeto atual (ao contrário das tabelas)
Mobile-first
Codificar (e criar o design) iniciando pela versão para mobile (a com mais limitações) e adicionando visuais e interações para resoluções maiores. Usar essa técnica reduz MUITO a quantidade de CSS escrito nas media-queryes
Progressive enhancement
Primeiro o conteúdo (texto), depois o resto. Após colocar todo o conteúdo na página, começamos a fazer o CSS (visual), e por último adicionamos a interação (JS). Essa técnica vai ao encontro do mobile-first e do graceful degradation.
Graceful degradation
Preocupar-se com o estado da página se as imagens não carregarem, as fontes falharem, o JS não carregar ou certo componente não estar presente no navegador, o CSS não carregar adequadamente ou certas regras não estarem presentes no navegador. Sempre tentando manter o máximo de usabilidade e apresentar o conteúdo mesmo assim.
reset/normalize/shiv CSS
Resetar e/ou normalizar o CSS para que todos os elementos apresentem um estilo inicial consistente entre os diferentes navegadores e dispositivos. É o primeiro (e um dos mais importantes) passos para o desenvolvimento responsivo com sucesso, pois evita as inconsistências inirtentes às diferentes plataformas.
