Condições

Essa é realmente a parte interessante da media-query, pois aqui conseguimos definir as condições para a aplicação de um determinado conjunto de regras CSS. Essa parte é opcional, mas, praticamente não faz sentido criar uma media-query sem condições. O mais comum é utilizarmos uma regra com AND e aí encadear o que precisamos.

MOBILE TABLET DESKTOP

Exemplo de uma condição em uma media-query

                
           +-------------------------+        
@media all | and (min-width: 1025px) |
{          +-------------------------+

    /*  REGRAS  */
    
}

é equivalente à

           +------------------------+        
@media all | and (width >= 1025px) |
{          +------------------------+

    /*  REGRAS  */
    
}

Condições mais utilizadas

min-width
Largura mínima da tela, desenvolvendo mobile-first essa é de longe a condição mais comum a ser utilizada nas tuas media-queryes.
min-height
Altura mínima da janela do navegador.
max-width
Largura máxima, geralmente visto quando há uma alteração / correção que visa apenas o mobile.
max-height
Altura máxima da janela do navegador.
min-device-width e max-device-width
Dimensões da TELA do dispositivo, são CONSTANTES, logo, se há janelas de tamanho alterável (como no desktop) essa condição pode não ser o que tu precisas.
orientation
portrait ou landscape , serve para checar a orientação do dispositivo, útil para trocar alguns hooks de tamanho da largura para a altura.
min-device-aspect-ratio, e max-device-aspect-ratio
A razão entre a largura e a altura da tela, como em 4:3, 16:9, etc ...
min-aspect-ratio, aspect-ratio, e max-aspect-ratio
A razão entre a largura e a altura da viewport (área visível / janela)
min-resolution, resolution, e max-resolution
Resolução do DISPOSITIVO (geralmente medida em dpi - dots per inch), usado, monitores geralmente apresentam 96dpi, telas retina 192, smartphones sem tela retina repousam entre esses valores, e valores mais altos só são comuns em mídia impressa. Juntos o webkit-min-device-pixel-ratio e o min-resolution servem para criar uma media-query exclusiva para telas retina.
-webkit-min-device-pixel-ratio
Pixel ratio, quantos, pixeis físicos fazem um pixel lógico. 1 para monitores, 2 para displays retina e alguns valores intermediários para smartphones sem tela retina.
Media queryes exclusivas para algumas telas mobile

Não esqueçamos que podemos usar os operadores , (OR), AND e NOT, e na dificuldade podemos ainda fazer uma media-query DENTRO de outra media-querye. 

                
@media screen and (max-width: 320px) 
              and (max-height: 568px)
{
	
	/*  REGRAS para o iPhone 5  */
	
}

é equivalente a 

@media screen and (max-width: 320px) 
{
	
	@media screen and (max-height: 568px)
	{
		
		/*  REGRAS para o iPhone 5  */
	
	}
	
}
