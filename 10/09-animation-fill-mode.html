animation-fill-mode

Que estado o elemento ficará quando a animação terminar, portanto é mais relevante para animações não infinitas. Por padrão o valor none não altera o estado do elemento, ou seja, assim que a animação termina ele volta para o estado que estava antes da animação iniciar. Mas temos a opção de setando um animation-fill de escolher o estado final do elemento. AS opções são

none
loading‑15
forwards
loading‑16
backwards
loading‑17
both
loading‑18
 #loading-15 .inner-bar {
	animation:          loading-4 2s ease 0s 1 normal none running;
	-webkit-animation:  loading-4 2s ease 0s 1 normal none running;
}

#loading-16 .inner-bar {
	animation:          loading-4 2s ease 0s 1 normal forwards running;
	-webkit-animation:  loading-4 2s ease 0s 1 normal forwards running;
}

#loading-17 .inner-bar {
	animation:          loading-4 2s ease 0s 1 normal backwards running;
	-webkit-animation:  loading-4 2s ease 0s 1 normal backwards running;
}

#loading-18 .inner-bar {
	animation:          loading-4 2s ease 0s 1 normal both running;
	-webkit-animation:  loading-4 2s ease 0s 1 normal both running;
}
